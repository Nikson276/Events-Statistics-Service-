
# Схемы Данных Событий Kafka

## Event Schema (Kafka Message Payload)

| Поле        | Тип      | Обязательное | Описание                          | Пример                     |
|-------------|----------|--------------|-----------------------------------|----------------------------|
| `id`        | string   | да           | Уникальный ID события             | `"evt_12345"`             |
| `user_id`   | string   | да           | ID пользователя                   | `"usr_67890"`             |
| `track_id`  | string   | да           | ID трека                          | `"trk_abcde"`             |
| `timestamp` | datetime | да           | Время события в UTC (ISO 8601)    | `"2025-11-30T12:00:00Z"`  |

### Ограничения

- Все ID — строки (не числа!), чтобы поддерживать UUID или хеши.
- timestamp всегда в UTC, с временной зоной.

### Эволюция схемы

- Новые поля — опциональные.
- Удаление полей — только после миграции и согласования.
- Никаких breaking changes без версионирования (если потребуется — заведём event_type и schema_version)
